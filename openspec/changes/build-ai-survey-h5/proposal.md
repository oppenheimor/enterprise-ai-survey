## Why

当前仓库只有早期脚手架页面，尚未把 `docs/requirements` 中已经收敛完成的企业 AI 转型评估 H5 主流程真正落到产品规格上。这个项目同时涉及用户行为、问卷流程、评分逻辑、LLM 输出边界、留资落库和运营看板，继续跳过规格直接开写，只会把后面的返工成本养大。

本次需要先把唯一主方案固化成可执行 change：以 `16 题逐题页` 为默认交互，以 `Bento Grid` 为主视觉方向，用结构化评分先算结论，再由 LLM 只做解释层分析，并把最小运营看板的数据链路一起纳入范围。

## What Changes

- 新增企业 AI 转型评估 H5 的前台主流程规格，覆盖落地页、16 题问卷、结果预览、留资页和成功页。
- 新增共享问卷配置与评分规则规格，要求前后端基于同一份题目、选项、分值和推荐场景映射工作。
- 新增 LLM 结果生成规格，要求输入必须结构化、输出必须可回溯、不得编造外部事实或 ROI。
- 新增线索落库与最小运营看板规格，确保问卷答案、评分结果、LLM 输出摘要与留资信息能被统一查询和跟进。
- 明确本次 UI 方向采用 `Bento Grid`，但不把“风格探索墙”继续扩展成新的用户流程。
- 明确验证基线：默认执行 `pnpm check` 与 `pnpm spec:validate`；涉及核心流程时补充规则测试、结果 benchmark 与 e2e。

## Capabilities

### New Capabilities
- `survey-h5-experience`: 提供面向企业负责人的移动端 H5 主流程，采用 Bento Grid 视觉语言承载落地页、16 题逐题问卷、结果预览、留资和成功反馈。
- `survey-scoring-engine`: 提供前后端共享的问卷配置、评分常量、评级规则、主推荐场景和线索优先级计算能力。
- `llm-diagnosis-output`: 基于结构化问卷答案与评分结果生成克制、可解释、可验证的 AI 转型初步诊断输出。
- `survey-lead-ops-dashboard`: 保存问卷与线索数据，并提供最小运营看板查看转化、评分分布和线索列表。

### Modified Capabilities
- None.

## Impact

- 影响代码范围：`app/` 路由、问卷与结果组件、服务端 actions 或 route handlers、共享配置模块、测试用例、Prisma schema。
- 影响数据模型：需要为问卷结果、留资信息、评分摘要、LLM 输出摘要与运营看板查询准备持久化结构。
- 影响验证体系：需要新增评分回归测试、LLM benchmark 执行方式，以及覆盖主流程的 e2e 或 smoke 验证。
- 外部依赖暂不新增复杂第三方平台；LLM 集成以受控输出协议为前提，避免把模型调用直接扩散进前端页面逻辑。
