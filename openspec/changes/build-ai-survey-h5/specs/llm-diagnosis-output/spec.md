## ADDED Requirements

### Requirement: 结构化输入协议
系统 SHALL 在调用 LLM 前构造结构化输入，其中 MUST 包含企业画像、16 题答案、评分结果与生成约束。

#### Scenario: 服务端准备 LLM 输入
- **WHEN** 用户完成问卷且规则评分已完成
- **THEN** 系统向 LLM 传入结构化对象
- **AND** 输入中包含问卷答案、评分摘要、主推荐场景和禁止项约束

### Requirement: 结构化诊断输出
系统 SHALL 要求 LLM 返回可渲染的结构化结果，至少覆盖当前阶段判断、核心问题总结、推荐起点、证据链、当前最大阻力、下一步建议、风险提醒、可信度说明和边界说明。

#### Scenario: LLM 返回诊断结果
- **WHEN** 系统收到模型输出
- **THEN** 输出可被解析为预定义 schema
- **AND** 前端结果页根据结构化字段渲染，而不是直接展示原始自由文本

### Requirement: 输出必须服从评分与边界约束
系统 MUST 阻止 LLM 输出与评分冲突、编造外部事实、编造具体 ROI 数字或缺少边界说明的结果进入正式页面。

#### Scenario: 模型输出与评分冲突
- **WHEN** 模型输出的结论与 `score_rating`、`top_scene` 或其他结构化评分结果矛盾
- **THEN** 系统将该结果视为失败
- **AND** 不得把冲突结果作为正式结果页内容

#### Scenario: 模型输出越界
- **WHEN** 模型输出出现行业数据、市场数据、收益承诺或缺失边界说明
- **THEN** 系统将该结果视为失败
- **AND** 该失败会触发阻断而不是静默放行

### Requirement: Benchmark 回归验证
系统 SHALL 支持使用 benchmark 样本与验收标准对 LLM 结果做回归验证，并将硬阻断项作为上线前门槛。

#### Scenario: 执行 benchmark 样本回归
- **WHEN** 团队修改 prompt、模型接入方式或评分规则
- **THEN** 系统可对预置样本执行结果验证
- **AND** 任一样本出现幻觉事实、评分冲突或无边界说明时视为失败
