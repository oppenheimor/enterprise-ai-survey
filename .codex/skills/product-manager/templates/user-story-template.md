# 用户故事模板

## 用户故事格式

```
作为一个 <用户角色>
我希望 <完成什么操作>
以便 <获得什么价值>
```

---

## 用户故事示例

### US-001 用户登录

**用户故事**
作为一个 未登录用户
我希望 能够使用账号密码登录系统
以便 访问我的个人数据和功能

**验收标准 (AC)**
- [ ] Given 用户在登录页面 When 输入正确的账号密码并点击登录 Then 成功进入首页
- [ ] Given 用户在登录页面 When 输入错误的密码 Then 显示"密码错误"提示
- [ ] Given 用户在登录页面 When 账号不存在 Then 显示"账号不存在"提示
- [ ] Given 用户已登录 When 关闭浏览器后重新打开 Then 保持登录状态

**故事点**: 3

**优先级**: P0

---

### US-002 数据搜索

**用户故事**
作为一个 系统用户
我希望 能够通过关键词搜索数据
以便 快速找到我需要的信息

**验收标准 (AC)**
- [ ] Given 用户在列表页面 When 输入关键词并搜索 Then 显示匹配的结果列表
- [ ] Given 搜索结果为空 When 搜索完成 Then 显示"暂无数据"提示
- [ ] Given 用户输入搜索词 When 按下回车键 Then 触发搜索

**故事点**: 2

**优先级**: P1

## 用户故事拆分原则 (INVEST)

| 原则 | 说明 | 检查项 |
|------|------|--------|
| **I**ndependent | 独立的 | 故事之间没有依赖关系 |
| **N**egotiable | 可协商的 | 细节可以与开发团队讨论 |
| **V**aluable | 有价值的 | 对用户或业务有明确价值 |
| **E**stimable | 可估算的 | 开发团队能够估算工作量 |
| **S**mall | 足够小的 | 一个迭代内可以完成 |
| **T**estable | 可测试的 | 有明确的验收标准 |

---

## 故事点估算参考

| 故事点 | 复杂度 | 工作量参考 |
|--------|--------|------------|
| 1 | 极简单 | 配置修改、文案调整 |
| 2 | 简单 | 简单UI、基础CRUD |
| 3 | 中等 | 常规功能、标准交互 |
| 5 | 复杂 | 复杂逻辑、多系统集成 |
| 8 | 很复杂 | 大型功能、架构调整 |
| 13+ | 需拆分 | 建议拆分为多个故事 |

---

## 优先级定义

| 优先级 | 定义 | 说明 |
|--------|------|------|
| P0 | 必须有 | MVP核心功能，缺失则无法上线 |
| P1 | 应该有 | 重要功能，显著影响用户体验 |
| P2 | 可以有 | 优化功能，提升用户体验 |
| P3 | 以后做 | 低优先级，后续版本考虑 |

## 验收标准编写指南

### Given-When-Then 格式

```
Given <前置条件/初始状态>
When <用户操作/触发事件>
Then <预期结果/系统响应>
```

### 示例

```
Given 用户已登录且有编辑权限
When 用户点击"编辑"按钮修改内容并保存
Then 系统保存修改并显示"保存成功"提示
```

## 用户故事工作流

```
新建 → 待评审 → 待开发 → 开发中 → 待测试 → 测试中 → 完成
  ↓        ↓         ↓         ↓         ↓
 废弃    需修改    阻塞     待验收    有缺陷
```
