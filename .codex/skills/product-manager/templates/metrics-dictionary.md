## 一、指标分类体系

### 1.1 指标层级结构

```
北极星指标 (North Star Metric)
    │
    ├── 一级指标 (Level 1 Metrics)
    │       │
    │       ├── 二级指标 (Level 2 Metrics)
    │       │       │
    │       │       └── 三级指标 (Level 3 Metrics)

```

### 1.2 指标类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 计数型 | 累计数量 | DAU、订单数 |
| 比率型 | 两个数值的比值 | 转化率、留存率 |
| 平均型 | 总量除以数量 | ARPU、平均时长 |
| 指数型 | 综合评分 | NPS、满意度指数 |

### 2.2 用户增长指标

#### DAU (Daily Active Users)

| 项目 | 内容 |
|------|------|
| **定义** | 日活跃用户数，当日登录/使用产品的独立用户数 |
| **计算公式** | COUNT(DISTINCT user_id) WHERE date = today AND action IN (登录, 核心行为) |
| **数据来源** | 埋点系统、用户行为日志 |
| **更新频率** | 每日 |
| **维度拆分** | 新用户DAU、老用户DAU、渠道DAU |
| **关联指标** | MAU、DAU/MAU比率、次日留存 |

#### MAU (Monthly Active Users)

| 项目 | 内容 |
|------|------|
| **定义** | 月活跃用户数，近30天内登录/使用产品的独立用户数 |
| **计算公式** | COUNT(DISTINCT user_id) WHERE date BETWEEN today-30 AND today |
| **数据来源** | 埋点系统、用户行为日志 |
| **更新频率** | 每日 |
| **维度拆分** | 新用户MAU、老用户MAU |

#### 新增用户数

| 项目 | 内容 |
|------|------|
| **定义** | 首次注册/激活的用户数量 |
| **计算公式** | COUNT(DISTINCT user_id) WHERE first_visit_date = today |
| **数据来源** | 用户注册系统 |
| **更新频率** | 每日 |
| **维度拆分** | 渠道来源、设备类型、地域 |

#### 用户留存率

| 项目 | 内容 |
|------|------|
| **定义** | 某批用户在后续时间点仍活跃的比例 |
| **计算公式** | 留存用户数 / 初始用户数 × 100% |
| **数据来源** | 用户行为日志 |
| **更新频率** | 每日 |
| **常见周期** | 次日留存、7日留存、30日留存 |

### 2.3 业务转化指标

#### 转化率 (Conversion Rate)

| 项目 | 内容 |
|------|------|
| **定义** | 完成目标行为的用户占总用户的比例 |
| **计算公式** | 完成转化的用户数 / 进入漏斗的用户数 × 100% |
| **数据来源** | 埋点系统 |
| **更新频率** | 实时 |
| **常见类型** | 注册转化率、购买转化率、支付转化率 |

#### 漏斗转化率

| 项目 | 内容 |
|------|------|
| **定义** | 用户在多步骤流程中各环节的转化比例 |
| **计算公式** | 当前步骤用户数 / 上一步骤用户数 × 100% |
| **数据来源** | 埋点系统 |
| **更新频率** | 实时 |
| **示例漏斗** | 首页→详情页→加购→下单→支付 |

#### GMV (Gross Merchandise Volume)

| 项目 | 内容 |
|------|------|
| **定义** | 商品交易总额（不扣除退款） |
| **计算公式** | SUM(order_amount) WHERE status IN (已支付, 已完成) |
| **数据来源** | 订单系统 |
| **更新频率** | 实时 |
| **维度拆分** | 品类、渠道、地域 |

### 2.4 用户价值指标

#### ARPU (Average Revenue Per User)

| 项目 | 内容 |
|------|------|
| **定义** | 每用户平均收入 |
| **计算公式** | 总收入 / 活跃用户数 |
| **数据来源** | 支付系统、用户系统 |
| **更新频率** | 每日 |
| **维度拆分** | 新用户ARPU、老用户ARPU |

#### ARPPU (Average Revenue Per Paying User)

| 项目 | 内容 |
|------|------|
| **定义** | 每付费用户平均收入 |
| **计算公式** | 总收入 / 付费用户数 |
| **数据来源** | 支付系统 |
| **更新频率** | 每日 |

#### LTV (Life Time Value)

| 项目 | 内容 |
|------|------|
| **定义** | 用户生命周期价值 |
| **计算公式** | ARPU × 用户生命周期长度 |
| **数据来源** | 支付系统、用户行为日志 |
| **更新频率** | 每周 |
| **计算方法** | 历史数据拟合、预测模型 |

#### CAC (Customer Acquisition Cost)

| 项目 | 内容 |
|------|------|
| **定义** | 用户获取成本 |
| **计算公式** | 总营销费用 / 新增用户数 |
| **数据来源** | 财务系统、营销系统 |
| **更新频率** | 每月 |
| **维度拆分** | 渠道CAC、自然流量CAC |

### 2.5 用户满意度指标

#### NPS (Net Promoter Score)

| 项目 | 内容 |
|------|------|
| **定义** | 净推荐值，衡量用户忠诚度 |
| **计算公式** | 推荐者比例 - 贬损者比例 |
| **数据来源** | 用户调研 |
| **更新频率** | 每月/每季度 |
| **评分标准** | 9-10推荐者, 7-8中立者, 0-6贬损者 |

#### CSAT (Customer Satisfaction Score)

| 项目 | 内容 |
|------|------|
| **定义** | 客户满意度评分 |
| **计算公式** | 满意用户数 / 调研用户数 × 100% |
| **数据来源** | 用户调研 |
| **更新频率** | 实时 |
| **评分标准** | 1-5分，4-5分为满意 |

#### CES (Customer Effort Score)

| 项目 | 内容 |
|------|------|
| **定义** | 客户费力指数，衡量服务易用性 |
| **计算公式** | 用户评分的平均值 |
| **数据来源** | 用户调研 |
| **更新频率** | 实时 |
| **评分标准** | 1-7分，分数越低越好 |

### 2.6 产品使用指标

#### 人均使用时长

| 项目 | 内容 |
|------|------|
| **定义** | 用户平均每次/每日使用产品的时间 |
| **计算公式** | SUM(session_duration) / session_count |
| **数据来源** | 埋点系统 |
| **更新频率** | 每日 |
| **维度拆分** | 新用户、老用户、不同页面 |

#### 人均使用次数

| 项目 | 内容 |
|------|------|
| **定义** | 用户平均每日打开/使用产品的次数 |
| **计算公式** | 总使用次数 / 活跃用户数 |
| **数据来源** | 埋点系统 |
| **更新频率** | 每日 |

#### 功能使用率

| 项目 | 内容 |
|------|------|
| **定义** | 使用某功能的用户占总用户的比例 |
| **计算公式** | 使用功能的用户数 / 活跃用户数 × 100% |
| **数据来源** | 埋点系统 |
| **更新频率** | 每日 |

## 四、指标计算规则

### 4.1 去重规则

| 场景 | 规则 |
|------|------|
| 用户数统计 | 按user_id去重，无user_id按device_id去重 |
| 跨平台用户 | 同一user_id在多端只计一次 |
| 时间范围 | 按自然日/周/月计算，时区统一为UTC+8 |

### 4.2 异常数据处理

| 异常类型 | 处理方式 |
|----------|----------|
| 数据缺失 | 使用前一日数据填充或标记为N/A |
| 数据异常波动 | 人工确认后决定是否剔除 |
| 重复数据 | 去重处理 |

### 4.3 指标口径变更

| 变更类型 | 处理方式 |
|----------|----------|
| 计算公式变更 | 需同步历史数据，保持可比性 |
| 数据来源变更 | 需进行数据对齐验证 |
| 维度调整 | 需更新相关文档和看板 |
