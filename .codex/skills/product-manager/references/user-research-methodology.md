## 一、调研方法概览

### 1.1 定性 vs 定量

| 类型 | 方法 | 特点 | 适用场景 |
|------|------|------|----------|
| **定性研究** | 用户访谈、焦点小组、可用性测试 | 深度理解"为什么" | 探索新方向、理解用户动机 |
| **定量研究** | 问卷调查、数据分析、A/B测试 | 大规模验证"是什么" | 验证假设、量化用户行为 |

### 1.2 方法选择矩阵

| 调研目标 | 推荐方法 | 样本量 | 周期 |
|----------|----------|--------|------|
| 探索用户需求 | 用户访谈 | 5-10人 | 1-2周 |
| 验证产品概念 | 焦点小组 | 2-3组(每组6-8人) | 2周 |
| 评估易用性 | 可用性测试 | 5-8人 | 1周 |
| 量化用户特征 | 问卷调查 | 200+人 | 2-3周 |
| 优化设计方案 | A/B测试 | 流量分流 | 1-2周 |

## 二、用户访谈

### 2.1 访谈类型

| 类型 | 说明 | 时长 | 适用场景 |
|------|------|------|----------|
| 深度访谈 | 一对一深入交流 | 45-60分钟 | 探索用户需求、动机 |
| 半结构化访谈 | 有提纲但灵活 | 30-45分钟 | 收集特定信息 |
| 情境访谈 | 在用户实际环境中 | 60-90分钟 | 理解真实使用场景 |

### 2.2 访谈准备清单

#### 访谈前
- [ ] 明确访谈目标（要了解什么）
- [ ] 确定目标用户画像
- [ ] 招募符合条件的用户
- [ ] 准备访谈提纲
- [ ] 预约访谈时间和地点
- [ ] 准备录音/录像设备
- [ ] 准备礼品/报酬

#### 访谈提纲结构
```
1. 开场（5分钟）
   - 自我介绍
   - 说明访谈目的
   - 确认录音许可
   - 建立信任关系

2. 背景了解（10分钟）
   - 基本信息
   - 相关经验
   - 使用习惯

3. 核心问题（30分钟）
   - 按主题展开
   - 深挖细节
   - 追问"为什么"

4. 总结收尾（5分钟）
   - 补充问题
   - 感谢致谢
```

### 2.3 访谈技巧

#### 提问技巧
| 技巧 | 示例 | 说明 |
|------|------|------|
| 开放式提问 | "您平时是怎么处理这个问题的？" | 避免是/否回答 |
| 追问 | "能具体说说吗？"、"为什么这样觉得？" | 深挖细节 |
| 澄清 | "您的意思是...对吗？" | 确认理解 |
| 情境假设 | "如果...您会怎么做？" | 探索潜在需求 |

#### 避免的错误
| 错误 | 问题 | 正确做法 |
|------|------|----------|
| 引导性提问 | "您觉得这个功能好用吗？" | "您对这个功能有什么感受？" |
| 假设性提问 | "您使用这个功能时..." | "您是否使用过这个功能？" |
| 连续提问 | 一次问多个问题 | 一次只问一个问题 |
| 评价用户 | "这个想法很好" | 保持中立态度 |

### 2.4 访谈记录模板

```
访谈编号：I-001
访谈时间：{日期} {时间}
访谈地点：{地点/线上}
受访者信息：
  - 编号：U-001
  - 性别：{性别}
  - 年龄：{年龄段}
  - 职业：{职业}
  - 使用频率：{频率}

访谈记录：
【问题1】{问题内容}
  - 回答：{用户回答}
  - 观察：{用户表情、动作等}
  - 追问：{追问内容}
  - 洞察：{关键发现}

【问题2】...

关键发现：
1. {发现1}
2. {发现2}
3. {发现3}

待验证假设：
- {假设1}
- {假设2}
```

## 三、问卷调查

### 3.1 问卷设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| 目标明确 | 每个问题都有目的 | 删除无关问题 |
| 简洁高效 | 控制问卷长度 | 核心问题不超过20个 |
| 逻辑清晰 | 问题有序排列 | 从简单到复杂 |
| 中立客观 | 避免引导性 | 使用中性表述 |
| 易于回答 | 减少用户负担 | 多用选择题 |

### 3.2 问题类型选择

| 类型 | 适用场景 | 示例 |
|------|----------|------|
| 单选题 | 互斥选项 | 您的性别是？ |
| 多选题 | 多种可能 | 您使用过哪些功能？ |
| 李克特量表 | 态度测量 | 您对产品的满意度（1-5分） |
| 评分题 | 程度评估 | 请给这个功能打分（1-10分） |
| 排序题 | 优先级判断 | 请按重要性排序 |
| 开放题 | 收集建议 | 您有什么改进建议？ |

### 3.3 问卷结构模板

```
【问卷标题】{简洁明确的标题}

【问卷说明】
感谢您参与本次调研！本问卷约需5分钟，您的回答将帮助我们改进产品。所有信息仅用于研究目的，我们将严格保密。

【第一部分：筛选问题】（1-2题）
用于筛选目标用户

【第二部分：基本信息】（3-5题）
- 性别
- 年龄
- 职业
- 使用频率
- 使用时长

【第三部分：核心问题】（10-15题）
按主题分组，从简单到复杂

【第四部分：开放问题】（1-2题）
- 您最满意/不满意的是什么？
- 您有什么建议？

【结束语】
感谢您的参与！如有疑问请联系xxx
```

### 3.4 问卷质量检查

| 检查项 | 说明 |
|--------|------|
| 逻辑跳转 | 条件跳转是否正确 |
| 必填设置 | 核心问题是否设为必填 |
| 选项完整 | 是否包含"其他"选项 |
| 量表一致 | 评分方向是否一致 |
| 预测试 | 小范围测试是否顺畅 |

## 四、可用性测试

### 4.1 测试类型

| 类型 | 说明 | 适用阶段 |
|------|------|----------|
| 形成性测试 | 发现问题、指导改进 | 设计/开发阶段 |
| 总结性测试 | 评估整体效果 | 上线前后 |
| 对比测试 | 比较不同方案 | 方案选择 |
| 远程测试 | 线上进行 | 用户分散时 |

### 4.2 测试流程

```
准备阶段 → 招募用户 → 执行测试 → 分析报告
```

#### 准备阶段
- [ ] 确定测试目标
- [ ] 设计测试任务
- [ ] 准备测试原型/产品
- [ ] 编写测试脚本
- [ ] 设置测试环境
- [ ] 准备记录表格

#### 测试任务设计
| 要素 | 说明 | 示例 |
|------|------|------|
| 场景 | 设定情境 | "假设您想购买一件商品..." |
| 任务 | 具体操作 | "请找到并加入购物车" |
| 成功标准 | 完成定义 | 成功加入购物车 |
| 时间限制 | 可选 | 建议不超过5分钟 |

### 4.3 测试脚本模板

```
【开场介绍】（5分钟）
"感谢您参与今天的测试。我们正在测试产品，不是测试您。过程中请大声说出您的想法，如果有任何困惑或问题，请随时告诉我。整个过程大约需要30分钟，我们会录音以便后续分析，您同意吗？"

【前置访谈】（5分钟）
- 您使用过类似产品吗？
- 您平时如何完成这类任务？

【任务1】（5分钟）
场景：{场景描述}
任务：{任务描述}
（观察记录：操作步骤、犹豫点、错误、言语表达）

【任务后提问】
- 刚才那个任务您觉得怎么样？
- 有什么地方让您困惑吗？
- 您觉得哪里可以改进？

【任务2】...

【整体反馈】（5分钟）
- 整体感受如何？
- 您最喜欢/不喜欢什么？
- 有什么建议？

【结束致谢】
感谢您的参与！
```

### 4.4 观察记录表

| 维度 | 记录内容 |
|------|----------|
| 任务完成 | 完成/部分完成/失败 |
| 完成时间 | 实际用时 |
| 错误次数 | 操作错误次数 |
| 求助次数 | 询问帮助次数 |
| 困惑表现 | 犹豫、停顿、迷茫 |
| 情绪反应 | 沮丧、惊喜、困惑 |
| 用户原话 | 关键表述记录 |

### 4.5 问题严重程度评级

| 级别 | 定义 | 处理优先级 |
|------|------|------------|
| **致命** | 用户无法完成任务 | 必须修复 |
| **严重** | 用户严重受阻，需要帮助 | 高优先级 |
| **中等** | 用户遇到明显困难但能完成 | 中优先级 |
| **轻微** | 小的困扰或不便 | 低优先级 |
| **建议** | 优化建议 | 按需处理 |

## 五、焦点小组

### 5.1 焦点小组特点

| 优势 | 局限 |
|------|------|
| 获得多元观点 | 可能被主导意见影响 |
| 互动产生新想法 | 不适合敏感话题 |
| 效率较高 | 不适合深度探索 |
| 观察群体动态 | 结果难以量化 |

### 5.2 组织要点

| 要素 | 建议 |
|------|------|
| 人数 | 6-8人/组 |
| 时长 | 90-120分钟 |
| 场次 | 2-4组（不同用户群） |
| 主持人 | 中立、善于引导 |
| 环境 | 舒适、无干扰 |

### 5.3 讨论提纲结构

```
1. 开场介绍（10分钟）
   - 规则说明
   - 破冰活动

2. 主题讨论（60分钟）
   - 话题1（20分钟）
   - 话题2（20分钟）
   - 话题3（20分钟）

3. 总结收尾（20分钟）
   - 补充讨论
   - 优先级排序
   - 感谢致谢
```

## 六、数据分析方法

### 6.1 定性数据分析

#### 编码分析法
```
1. 熟悉数据：阅读所有访谈记录
2. 初步编码：标记关键语句
3. 归类整理：将编码归类为主题
4. 提炼洞察：总结核心发现
5. 验证结论：回溯原始数据验证
```

#### 亲和图法
```
1. 将用户反馈写在卡片上
2. 将相似内容的卡片归类
3. 为每个类别命名
4. 分析各类别的关系
5. 提炼关键洞察
```

### 6.2 定量数据分析

| 分析类型 | 方法 | 适用场景 |
|----------|------|----------|
| 描述性分析 | 频次、均值、百分比 | 了解基本情况 |
| 交叉分析 | 不同维度交叉 | 发现差异 |
| 相关分析 | 相关系数 | 发现关联 |
| 对比分析 | T检验、卡方检验 | 验证差异显著性 |

## 八、调研伦理

### 8.1 基本原则

| 原则 | 说明 |
|------|------|
| 知情同意 | 告知目的、过程、风险 |
| 隐私保护 | 匿名处理、数据保密 |
| 自愿参与 | 可随时退出 |
| 无伤害原则 | 避免造成心理/利益伤害 |

### 8.2 实践要点

- [ ] 获取用户书面/口头同意
- [ ] 说明数据用途和保存方式
- [ ] 提供联系方式供用户咨询
- [ ] 按承诺支付报酬/礼品
- [ ] 及时删除敏感信息
