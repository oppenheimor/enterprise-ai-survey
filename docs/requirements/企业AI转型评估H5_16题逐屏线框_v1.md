## 1. 说明

本文档将 16 题问卷拆到“逐屏”粒度。

默认交互模式：

1. 一屏一题
2. 顶部展示总进度和当前题号
3. 中部展示题目和选项
4. 底部展示“上一题 / 下一步”

说明：虽然 PRD 里按 4 个步骤分组，但具体交互落地建议采用 `16 题逐屏`，因为：

1. 手机端一屏一题更轻
2. 用户感知压力更低
3. 更利于埋点和中断恢复

## 4. 逐屏实现建议

1. 进度条统一显示 `X / 16`，比“第几步”更直观。
2. Q5 由于有 6 个选项，页面需支持竖向滚动但底部按钮保持可见。
3. 每题选择后可自动激活“下一步”，但不建议自动跳题。
4. 第 17 屏提交问卷时需要展示短暂 loading，避免用户误点。
5. 结果页的“为什么这么判断”建议固定 3 条证据，不要过长。
