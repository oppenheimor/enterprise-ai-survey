## 1. 文档目标

本文档用于定义本次 H5 问卷产品的唯一需求基线。

本版已明确收口为：

1. 主方案采用 `16 题问卷`。
2. 问卷 `全部为选择题`，不要求用户手填任何业务信息。
3. 问卷完成后先由结构化规则计算评分，再由 LLM 做解释层分析。

本文档不承载甲方内部 SOP、技术型智能体 2、报价体系、投资筛选和复杂实施方案。

---

## 2. 需求范围收敛

### 2.1 本次必须做的范围

1. 用户进入 H5 并完成 16 题企业转型评估问卷。
2. 系统基于规则计算结构化评分和推荐标签。
3. 系统基于结构化输入调用 LLM 生成结果分析。
4. 用户查看结果预览后完成留资。
5. 线索与问卷结果完整落库，供后续跟进。

### 2.2 本次明确不做的范围

1. 技术型智能体 2 的复杂表单、技术测试、投资评分和详细 ROI 建模。
2. 完整行业 benchmark 报告。
3. 精确收益承诺、实施报价和交付排期。
4. 复杂 CRM 工作台。
5. 小程序同步交付。

## 3. 产品背景与目标

### 3.1 问题背景

甲方希望通过一个低门槛但又不能太浅的 AI 转型诊断产品，帮助企业负责人快速判断：

1. 现在值不值得启动 AI 转型。
2. 如果启动，先从哪里开始最合理。
3. 当前最大的现实阻力是什么。
4. 下一步应该如何推进。

### 3.2 产品目标

1. 用 16 题均衡问卷完成第一轮有效筛选。
2. 用结构化评分 + LLM 分析建立专业感。
3. 用结果预览和完整报告权益提升留资率。
4. 给后续顾问跟进提供可解释的诊断基础。

### 3.3 成功指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| 问卷开始率 | 落地页访问后点击开始诊断的比例 | ≥ 55% |
| 问卷完成率 | 开始诊断后完成全部 16 题的比例 | ≥ 35% |
| 留资转化率 | 查看结果后提交联系方式的比例 | ≥ 30% |
| 页面加载时间 | 首屏可交互时间 | ≤ 3 秒 |
| 结果生成时间 | 提交问卷后结果页渲染完成时间 | ≤ 3 秒 |

## 4. 目标用户

### 4.1 核心用户画像

#### 用户类型 A：传统行业企业主

- 企业规模：10 至 200 人
- 行业：制造、零售、服务等传统行业
- 核心诉求：降本、提效、减少重复劳动、找到 AI 切入口
- 主要顾虑：投入高、效果不确定、员工不会用、数据安全

#### 用户类型 B：服务型公司负责人

- 企业规模：5 至 50 人
- 行业：咨询、培训、设计、代运营、律所等
- 核心诉求：提高人效、提升获客和交付效率、用 AI 替代低价值重复工作
- 主要顾虑：不知道从哪里开始、工具太多不会选、无法团队化落地

---

## 5. 产品定位

本产品是一个“企业 AI 转型评估入口型 H5”。

它不是：

1. 企业级 AI 实施平台
2. 技术可行性测试系统
3. ROI 计算器
4. 咨询顾问交付报告系统

它是：

1. 对外的转化入口
2. 对内的线索筛选工具
3. 对用户的初步判断系统

---

## 6. 信息架构

本次 H5 主流程共 8 个页面：

1. 落地页
2. 问卷步骤一：企业情况
3. 问卷步骤二：当前痛点
4. 问卷步骤三：基础与条件
5. 问卷步骤四：决策偏好
6. 结果预览页
7. 留资解锁页
8. 提交成功页

补充状态：

1. 加载中状态
2. 网络异常状态
3. 表单校验错误状态

---

## 7. 核心流程

1. 用户进入落地页，理解价值主张。
2. 用户点击“立即开始诊断”。
3. 用户完成 4 个步骤、16 道选择题问卷。
4. 系统计算结构化评分、评级、主推荐场景和线索优先级。
5. 系统基于结构化输入调用 LLM 生成结果分析。
6. 用户查看结果预览。
7. 用户提交联系方式以解锁完整结果权益。
8. 系统展示提交成功页。

---

## 8. 页面需求

### 8.1 页面一：落地页

#### 页面目标

在 3 秒内说明产品价值并驱动点击开始。

#### 必要内容

1. 主标题
2. 副标题
3. 价值点 3 条
4. 适用人群说明
5. 结果权益说明
6. 主 CTA

---

### 8.2 页面二至页面五：16 题问卷流程

#### 页面目标

在不要求用户手填业务信息的前提下，完成足够可信的初步诊断输入采集。

#### 题目结构

| 模块 | 题号 | 目标 |
|------|------|------|
| 企业情况 | Q1-Q4 | 判断行业、规模、业务类型、收入方式 |
| 当前痛点 | Q5-Q8 | 判断主场景、重复性、核心问题、影响程度 |
| 基础与条件 | Q9-Q14 | 判断 AI 基础、工具基础、团队接受度、预算、时间、负责人 |
| 决策偏好 | Q15-Q16 | 判断转型意愿和用户最想看到的结果类型 |

#### 关键约束

1. 16 题全部为选择题。
2. 不允许出现开放文本题。
3. 每一步必须能清楚看到进度。
4. 用户未选择答案时不能进入下一题。

---

### 8.3 页面六：结果预览页

#### 页面目标

用结构化评分 + LLM 解释生成一份可信、克制、可解释的初步结果。

#### 必要内容

1. 当前阶段判断
2. 核心问题归因
3. 优先切入方向
4. 为什么这么判断
5. 当前最大阻力
6. 下一步建议
7. 边界说明
8. 解锁完整结果 CTA

#### 关键约束

1. 不得输出未经输入支持的行业数据。
2. 不得输出未经验证的具体 ROI 数字。
3. 不得和评分结果冲突。

---

### 8.4 页面七：留资解锁页

#### 页面目标

完成线索收集。

#### 字段设计

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | 2-20 字 |
| 手机号 | 文本 | 是 | 中国大陆手机号格式 |
| 企业名称 | 文本 | 是 | 2-50 字 |
| 微信号/备用联系方式 | 文本 | 否 | 便于跟进 |
| 最关注的问题 | 单选 | 否 | 降本、提效、获客、管理、其他 |

说明：问卷禁止手填，不等于留资页禁止手填。联系方式字段仍需文本输入。

## 9. 评分与结果生成

### 9.1 评分规则

总分 100 分，由三部分组成：

1. 转型潜力：40 分
2. 痛点迫切度：30 分
3. 落地可行性：30 分

### 9.2 结果生成规则

系统采用两段式生成：

1. 规则引擎先计算：
   - 分数
   - 等级
   - 主推荐场景
   - 线索优先级
2. LLM 再生成：
   - 当前阶段判断
   - 核心问题总结
   - 推荐方向
   - 证据链
   - 风险提醒
   - 下一步动作

### 9.3 结果边界

允许输出：

1. 初步判断
2. 优先方向建议
3. 当前阻力判断
4. 小步推进建议

不允许输出：

1. 具体收益承诺
2. 行业 benchmark 断言
3. 完整技术实施方案
4. 报价和交付周期

### Must Have

1. 16 题问卷
2. 结构化评分
3. LLM 结果解释

## 13. 风险与待确认项

### 当前风险

1. 16 题比 7 题更重，完成率可能下降。
2. 如果结果页过度承诺，会损害可信度。
3. LLM 若无严格约束，容易输出模板化空话或越界结论。

### 当前结论

为平衡转化和诊断质量，本版本选择：

1. 保留 16 题
2. 问卷全部选择题
3. 评分规则先行，LLM 只做解释层
