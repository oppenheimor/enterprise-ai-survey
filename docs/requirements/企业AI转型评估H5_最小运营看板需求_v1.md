## 1. 文档目标

本看板用于承接 H5 问卷产生的数据，服务于运营、顾问和管理者的日常查看与跟进。

本次只做 `最小运营看板`，目标不是做完整 CRM，而是先解决三个实际问题：

1. 流量和转化是否正常。
2. 哪些线索值得优先跟进。
3. 当前用户集中在哪些行业、场景和评分区间。

---

## 2. 范围定义

### 2.1 本次必须做

1. 漏斗概览
2. 关键指标卡
3. 线索列表
4. 诊断结果分布
5. 基础筛选与搜索

### 2.2 本次明确不做

1. 完整 CRM 工作台
2. 顾问协作流程
3. 线索状态流转系统
4. 自动分配、自动催办
5. 权限细粒度体系
6. BI 级自由分析器

---

## 3. 目标用户

### 3.1 运营

关注：

1. 今天来了多少人
2. 转化有没有掉
3. 哪个渠道质量更高

### 3.2 顾问 / 销售

关注：

1. 哪些线索值得优先联系
2. 用户最关心什么
3. 用户更可能从哪个场景切入

### 3.3 管理者

关注：

1. 问卷整体完成情况
2. 留资效率
3. 高价值线索占比

---

## 4. 看板定位

本看板属于：

1. `运营日报看板`
2. `线索筛选看板`

不属于：

1. 实时监控大屏
2. 高层战略驾驶舱
3. 顾问实施后台

---

## 5. 页面结构

建议采用单页桌面端后台布局：

1. 顶部筛选区
2. 第一屏：关键指标卡 + 漏斗
3. 第二屏：评分 / 场景 / 行业分布
4. 第三屏：线索列表

---

## 6. 信息架构

### 模块 A：顶部筛选区

#### 目标

让用户可以快速缩小数据范围。

#### 筛选项

1. 时间范围
   - 今天
   - 最近 7 天
   - 最近 30 天
   - 自定义
2. 渠道来源
3. 行业类型
4. 企业规模
5. 主推荐场景
6. 评分等级
7. 线索等级

#### 搜索项

1. 企业名称
2. 手机号
3. 姓名

---

### 模块 B：关键指标卡

#### 目标

让用户 3 秒内看到当天或所选时间范围内的核心表现。

#### 指标卡清单

1. 落地页访问数
2. 开始诊断数
3. 完成问卷数
4. 留资数
5. 问卷完成率
6. 留资转化率
7. A/B 级线索数

#### 展示形式

每张卡显示：

1. 指标名称
2. 当前值
3. 与上个周期对比

---

### 模块 C：转化漏斗

#### 目标

直观看出流失点。

#### 漏斗定义

1. `landing_view`
2. `start_click`
3. `questionnaire_complete`
4. `result_view`
5. `lead_submit`

#### 展示要求

1. 显示每步人数
2. 显示步间转化率
3. 明确突出最大流失环节

---

### 模块 D：诊断结果分布

#### 目标

帮助运营和顾问理解当前问卷人群结构。

#### 图表清单

1. 行业分布
   - 图表：横向条形图
2. 主推荐场景分布
   - 图表：柱状图
3. 评分等级分布
   - 图表：环形图或柱状图
4. 线索等级分布
   - 图表：柱状图

#### 展示价值

1. 看哪个行业最多
2. 看用户最集中在哪类需求场景
3. 看高价值线索比例是否健康

---

### 模块 E：线索列表

#### 目标

让顾问直接拿到能跟进的线索清单。

#### 列表字段

1. 提交时间
2. 姓名
3. 手机号
4. 企业名称
5. 行业类型
6. 企业规模
7. 总分
8. 评分等级
9. 主推荐场景
10. 当前最大阻力
11. 线索等级
12. 最关注的问题

#### 行为

1. 支持按列排序：
   - 提交时间
   - 总分
   - 线索等级
2. 支持点击展开查看结果摘要

#### 结果摘要抽屉内容

1. 当前阶段判断
2. 核心问题总结
3. 推荐起点
4. 证据链
5. 下一步建议

## 7. 核心指标定义

| 指标 | 定义 |
|------|------|
| 落地页访问数 | 触发 `landing_view` 的去重会话数 |
| 开始诊断数 | 触发 `start_click` 的去重会话数 |
| 完成问卷数 | 触发 `questionnaire_complete` 的去重会话数 |
| 留资数 | 触发 `lead_submit` 的去重会话数 |
| 问卷完成率 | 完成问卷数 / 开始诊断数 |
| 留资转化率 | 留资数 / 查看结果数 或 留资数 / 完成问卷数 |
| 高价值线索数 | 线索等级为 A/B 的数量 |

说明：留资转化率分母建议在实现时统一为 `result_view`，不要一会儿除完成问卷，一会儿除开始诊断。

## 10. 非功能要求

| 类型 | 要求 | 优先级 |
|------|------|--------|
| 性能 | 首屏 3 秒内可展示指标卡和列表首屏数据 | P0 |
| 易用性 | 筛选条件变更后 2 秒内刷新结果 | P0 |
| 可读性 | 一屏内先看指标，再看分布，再看线索 | P0 |
| 可操作性 | 线索列表可按评分和等级排序 | P1 |
| 安全性 | 手机号默认脱敏展示，点击后再查看完整 | P0 |

---

## 11. MVP 验收标准

1. 用户可按时间范围查看访问、开始、完成、留资数据。
2. 漏斗可正确展示每步人数与转化率。
3. 可查看行业、场景、评分和线索等级分布。
4. 可查看线索列表，并按评分/等级排序。
5. 点击某条线索可查看结果摘要。
