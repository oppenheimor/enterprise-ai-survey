## 1. 文档目标

本文档用于把 Q1-Q16 每个选项的计分、标签和修正规则一次性写死，作为前后端共享的唯一常量源。

使用原则：

1. 前端展示和后端计算都基于同一套选项值。
2. 题目文案可微调，但选项值与分值不可随意漂移。
3. 如果修改任一题分值，必须同步修改评分映射表和 benchmark 样本。

---

## 2. 维度定义

总分 = 转型潜力 40 + 痛点迫切度 30 + 落地可行性 30

三个维度的原始分上限约定：

1. 转型潜力原始分上限：60
2. 痛点迫切度原始分上限：40
3. 落地可行性原始分上限：60

换算公式：

1. 转型潜力得分 = `min(40, round(raw_potential / 60 * 40))`
2. 痛点迫切度得分 = `min(30, round(raw_urgency / 40 * 30))`
3. 落地可行性得分 = `min(30, round(raw_feasibility / 60 * 30))`

## 4. 主推荐场景计算规则

### 4.1 基础场景

由 Q5 决定：

1. Q5=A -> 增长效率
2. Q5=B -> 客服效率
3. Q5=C -> 交付效率
4. Q5=D -> 内容效率
5. Q5=E -> 内部协同
6. Q5=F -> 通用优化

### 4.2 修正规则

1. 若 Q7=D，则在结果文案中提升“流程标准化”优先级。
2. 若 Q6=A 且 Q7=A，则在结果文案中提升“降本优化”强度。
3. 若 Q10=C/D，则在结果文案中增加“先补工具基础”提示。
